## CloudFront 배포 생성

- Distribution type: **Single website or app**
- Route 53 도메인: 연결 안 함 (보유 도메인 없음, 기본 `*.cloudfront.net` 사용)
- Origin access: **Origin access control (OAC)** — "Grant CloudFront access to origin: Yes" 선택 시 CloudFront가 S3 버킷 정책을 자동으로 갱신해줘서 수동 정책 복사 단계 불필요
- WAF: **보안 보호 비활성화** (월 ~$14 추가 비용 발생, 포트폴리오 프로젝트라 불필요)
- 생성된 도메인: `d3kvplpwaeztju.cloudfront.net`

### 콘솔 마법사 버전 차이
- 이번 콘솔 흐름에는 생성 단계 중 **Default root object 입력란이 없었음** (가이드 문서 기준과 다름)

## 트러블슈팅

### SSH 키 파일 권한 문제
- **증상**: Windows에서 다운로드한 `.pem` 키 파일은 기본적으로 권한이 너무 열려 있어 OpenSSH가 접속을 거부할 수 있음
- **해결**: `icacls`로 현재 사용자에게만 읽기 권한을 부여하도록 ACL 재설정
  ```powershell
  icacls learning-hub-key.pem /inheritance:r
  icacls learning-hub-key.pem /grant:r "$($env:USERNAME):(R)"
  ```

### 운영 RDS에 로컬 데이터가 없음
- **증상**: 배포 후 RDS에 로컬 개발 중 쌓은 기록(사용자, 게시글 등)이 보이지 않음
- **원인**: 로컬 PostgreSQL과 RDS는 완전히 분리된 별도 데이터베이스. `ddl-auto: update`로 스키마(테이블 구조)는 자동 생성되지만 데이터(row)는 별개로 마이그레이션해야 함
- **결론**: 의도된 정상 동작. 로컬 데이터를 가져오려면 `pg_dump`/`pg_restore`로 별도 이전 작업 필요 (이번엔 진행하지 않음 — 새 운영 환경은 깨끗한 상태로 시작)
