## Google Cloud Console OAuth2 설정

### 등록 절차
1. Google Cloud Console → 새 프로젝트 생성 (`learning-hub`)
2. API 및 서비스 → OAuth 동의 화면 → 외부 선택 → 앱 이름/이메일 입력
3. 사용자 인증 정보 → OAuth 2.0 클라이언트 ID 생성
   - 애플리케이션 유형: **웹 애플리케이션**
   - 승인된 리디렉션 URI: `http://localhost:8080/login/oauth2/code/google`
4. Client ID / Client Secret 발급

### 막힌 점 / 해결
- 첫 시도: `401 invalid_client` (The OAuth client was not found)
  → 리디렉션 URI를 등록하지 않았던 것이 원인, 추가 후 해결
- 두 번째 시도: `Invalid credentials` (Spring Security 에러 페이지)
  → `application-local.yml`이 로드되지 않고 있었음
  → `spring.profiles.active: local`을 `application.yml` 상단에 추가해서 해결
  → (⚠️ **배포 전 이 줄 반드시 삭제** — 운영 서버에서 local 프로파일 켜지면 앱 기동 실패)

## application-local.yml 설정

Spring Boot는 Node.js와 달리 `.env`를 자동으로 읽지 않음.
로컬 민감값은 `application-local.yml` (gitignore 등록)에서 관리.

```yaml
spring:
  security:
    oauth2:
      client:
        registration:
          google:
            client-id: {발급받은 ID}
            client-secret: {발급받은 Secret}
```

VSCode에서 local 프로파일로 실행하는 방법:
```bash
./mvnw spring-boot:run -Dspring-boot.run.profiles=local
```
또는 `application.yml` 상단에 `spring.profiles.active: local` 추가 (로컬 전용).

## Swagger UI 동작 검증

접속: `http://localhost:8080/swagger-ui/index.html`

### 검증 순서
1. **공개 API (인증 불필요)**
   - `GET /api/profiles/{slug}` → 404 응답 확인 (데이터 없음, 서버 정상 응답)

2. **JWT 토큰 발급**
   - 브라우저에서 `http://localhost:8080/oauth2/authorization/google` 접속
   - 구글 로그인 완료 → 프론트(localhost:3000) 없어서 연결 실패 페이지 뜸
   - 주소창 URL에서 `?token=` 뒤 값 복사

3. **인증 API 테스트**
   - Swagger UI 우상단 **Authorize 🔒** → 토큰 붙여넣기
   - `POST /api/records` 요청 → 201 Created 확인 ✅
   - `GET /api/records` 요청 → 생성된 기록 목록 확인 ✅
