## 방향 관련 결정

### 팀 플랫폼 → 개인 공개 프로필로 확정
"팀 학습 플랫폼"으로 리포지셔닝하는 아이디어를 검토했으나 기각.
이유 3가지:
1. 팀 도구는 팀 전체를 설득해야 해서 실사용자 확보가 오히려 더 어려워짐
2. 역할 권한/초대 등 구현 범위가 폭발
3. Confluence·Notion 등이 이미 장악한 시장

대신 `/@username` 공개 프로필 하나로 "포트폴리오"와 "실사용자" 두 목표를 동시에 해결.
→ 면접관에게 URL 하나로 "이 앱을 만들면서 이 앱으로 공부를 기록했다"는 것을 라이브로 증명 가능.

### 비밀번호 로그인 제거, 구글 소셜 로그인만 유지
- 비밀번호 방식: 회원가입 폼 + BCrypt + 검증 로직 필요
- 소셜 로그인만: "OAuth2 콜백 → JWT 발급"만 구현하면 끝, 범위가 더 작음
- 실사용자 관점에서도 소셜 로그인이 가입 장벽을 훨씬 낮춤

## 구현 내용

### 인증 레이어 (Google OAuth2 + JWT)
```
흐름: 프론트 → /oauth2/authorization/google
        → 구글 동의 화면
        → CustomOAuth2UserService (DB 유저 저장/조회)
        → OAuth2SuccessHandler (JWT 발급)
        → 프론트 /oauth2/callback?token=xxx
        → localStorage 저장
```

**주요 구성 파일**
- `JwtTokenProvider` — jjwt 0.12.6, HMAC-SHA 서명, subject에 userId 저장
- `JwtAuthFilter` — `OncePerRequestFilter` 상속, Bearer 토큰 파싱 → SecurityContext 설정
- `CustomOAuth2UserService` — 첫 로그인 시 자동 회원가입, `profileSlug` 자동 생성
- `OAuth2UserAdapter` — Spring의 `OAuth2User`와 도메인 `User`를 연결하는 어댑터
- `OAuth2SuccessHandler` — 성공 후 JWT를 쿼리 파라미터로 담아 프론트 리다이렉트
- `SecurityConfig` — CSRF 비활성화, Stateless 세션, CORS 설정, 권한 규칙

### 학습 기록 CRUD API
| 메서드 | 경로 | 설명 |
|---|---|---|
| POST | `/api/records` | 기록 생성 → 201 + Location 헤더 |
| GET | `/api/records` | 내 기록 목록 (페이지네이션, 최신순) |
| GET | `/api/records/{id}` | 단건 조회 |
| PUT | `/api/records/{id}` | 수정 |
| DELETE | `/api/records/{id}` | 삭제 → 204 |

### 공개 프로필 API (인증 불필요)
| 메서드 | 경로 | 설명 |
|---|---|---|
| GET | `/api/profiles/{slug}` | 닉네임, 슬러그, 공개 기록 수 |
| GET | `/api/profiles/{slug}/records` | 공개 기록 목록 (페이지네이션) |

## 적용한 디테일 (면접 어필 포인트)

- **`@Transactional(readOnly = true)`** 클래스 레벨 설정
  → 읽기 메서드는 flush 생략·스냅샷 미생성으로 성능 최적화, 쓰기 메서드만 `@Transactional` 오버라이드
- **`findByIdAndUser`** 소유권 검증
  → 다른 유저의 기록을 수정/삭제하지 못하도록 쿼리 레벨에서 차단 (서비스 레벨이 아닌 DB 레벨)
- **`replaceTags()` + `orphanRemoval`**
  → 태그 수정 시 `recordTags.clear()` → `addAll()` 패턴, orphanRemoval이 고아 RecordTag를 자동 삭제
- **fetch join 쿼리 (`@Query`)**
  → 공개 프로필 기록 조회 시 RecordTag + Tag를 한 번에 로딩해 N+1 방지
- **`ProfileSlug` 자동 생성 로직**
  → 구글 이름에서 영문/숫자만 추출 → 중복 시 UUID 6자리 추가 (유일성 보장)
- **Java record로 DTO 작성** (`LearningRecordRequest`, `LearningRecordResponse`, `ProfileResponse`)
  → 불변 객체, 보일러플레이트 최소화

## Swagger 연동
- `springdoc-openapi-starter-webmvc-ui 2.8.3` 추가
- `SwaggerConfig` — `@OpenAPIDefinition`(프로젝트 정보, 서버 목록), `@SecurityScheme`(JWT Bearer) 설정
- `SecurityConfig`에 `/swagger-ui/**`, `/v3/api-docs/**` permitAll 추가
- 각 컨트롤러에 `@Tag`, `@Operation`, `@ApiResponse`, `@SecurityRequirement` 어노테이션 작성
- 접속 주소: `http://localhost:8080/swagger-ui/index.html` → Authorize 버튼으로 JWT 입력 후 직접 테스트 가능

## 로컬 개발 환경 세팅

### Docker PostgreSQL
```bash
docker run --name learning-hub-db \
  -e POSTGRES_PASSWORD=postgres \
  -e POSTGRES_DB=learning_hub \
  -p 5432:5432 -d postgres:16

# Docker Desktop 재시작 후 자동 실행되도록
docker update --restart always learning-hub-db
```

### application-local.properties
- Spring Boot는 `.env`를 자동으로 읽지 않음 (Node.js와 다름)
- 로컬 민감값 관리 방법: `application-local.properties` + `.gitignore` 등록
- 실행 시 프로파일 활성화 필요: `./mvnw spring-boot:run -Dspring-boot.run.profiles=local`
- `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`은 이 파일에서 관리

### DDL 동작 방식 (`ddl-auto=update`)
- Docker 컨테이너 최초 생성 시 → `learning_hub` DB 생성
- 앱 최초 실행 시 → 테이블 5개 자동 생성
- 앱 재실행 시 → 엔티티 변경분만 반영 (컬럼 삭제는 건드리지 않음)
- `docker stop/start` → 데이터 보존 / `docker rm` → 데이터 소멸

## 막힌 점 / 해결
- Spring Initializr 기본 생성 시 `spring-boot-starter-webmvc`(Boot 4.x 아티팩트명)로 잡혀
  → `CustomOAuth2UserService`에서 `OAuth2UserAdapter` 미생성 상태로 컴파일 시도해 에러 발생
  → 참조 선언 전에 구현 클래스를 먼저 만들어야 한다는 걸 확인, 순서 조정으로 해결
- `application.properties` 민감값은 환경변수로 분리 (`${GOOGLE_CLIENT_ID}`, `${JWT_SECRET}` 등)
- 앱 첫 실행 시 `Connection to localhost:5432 refused` 에러
  → Docker Desktop이 꺼져 있었음, 실행 후 해결
  → IDE Swagger 어노테이션 에러는 pom.xml 변경 후 Maven 미리로드 때문인 false positive였음
