## 프론트엔드 세팅

### Tailwind v4 → v3 전환
- 초기에 Tailwind v4 (`@tailwindcss/vite` 플러그인 방식) 설치했으나 shadcn/ui CLI가 `tailwind.config.js` 없음 오류로 초기화 실패
- Tailwind v3으로 다운그레이드 후 `tailwind.config.js` + `postcss.config.js` 생성으로 해결
- shadcn/ui CLI도 React 19 peer dependency 문제로 대화형 프롬프트에서 막힘 → CLI 없이 컴포넌트 직접 구현 (`button`, `card`, `badge`)

### 프로젝트 구조
```
src/
├── api/          # axios 클라이언트 + API 함수
├── components/ui/ # shadcn 스타일 UI 컴포넌트
├── lib/          # cn() 유틸
├── pages/        # 페이지 컴포넌트
├── routes/       # 라우터 (index.tsx + PrivateRoute.tsx)
├── store/        # zustand 상태 관리
└── types/        # 공통 타입 정의
```

## 발생한 버그 & 해결

### 1. OAuth2 콜백이 localhost:3000으로 리다이렉트
- `application.yml`의 `app.frontend-url`이 `localhost:3000`으로 설정돼 있었음
- Vite 기본 포트는 5173 → `localhost:5173`으로 수정
- SecurityConfig CORS도 동일하게 수정 후 백엔드 재시작

### 2. ERR_TOO_MANY_REDIRECTS
- Vite 프록시에서 `/oauth2` 경로 전체를 백엔드로 포워딩
- `/oauth2/callback`(React 페이지)까지 백엔드로 넘어가 무한 리다이렉트 발생
- 프록시 설정을 `/oauth2/authorization`만 백엔드로 전달하도록 수정

```ts
// 수정 전
'/oauth2': 'http://localhost:8080'

// 수정 후
'/oauth2/authorization': 'http://localhost:8080',
'/login/oauth2': 'http://localhost:8080',
```

### 3. Vite 포트 자동 변경 (5173 → 5174)
- 이전에 실행한 Vite 프로세스가 백그라운드에 남아 5173 점유
- `vite.config.ts`에 `strictPort: true` 추가 → 포트 충돌 시 자동 변경 대신 에러로 명시

### 4. PrivateRoute 무한 API 호출
- zustand store의 `isAuthenticated` 함수를 의존성으로 사용 → 매 렌더마다 새 참조 생성
- `token` 값을 직접 읽도록 변경

### 5. `/records/new` 라우트 매칭 실패
- `/records/:id`가 `/records/new`보다 위에 정의돼 `id = "new"`로 잡힘
- `new` 라우트를 `:id` 위로 이동

### 6. `/@slug` 라우트 미매칭
- React Router가 `@` 포함 경로를 매칭 못 함 (Vite 내부 경로 `/@vite/client`와 충돌)
- `/profile/:slug`로 변경

### 7. 공개 프로필 `recordCount` undefined
- 백엔드 응답 필드명: `totalPublicRecords`
- 프론트 타입 필드명: `recordCount` (불일치)
- 프론트 타입을 `totalPublicRecords`로 수정

### 8. PageImpl 직렬화 경고
- `Serializing PageImpl instances as-is is not supported` WARN 발생
- `@EnableSpringDataWebSupport(pageSerializationMode = VIA_DTO)` 추가로 해결
- 프론트 `Page<T>` 타입의 페이지 메타 구조도 `page.totalElements` 형태로 업데이트
