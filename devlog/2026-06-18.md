## 트러블슈팅

### RDS 연결 타임아웃 (Connect timed out)
- **증상**: backend 컨테이너가 `SocketTimeoutException: Connect timed out`으로 기동 실패, Hibernate가 Dialect를 못 찾음
- **원인**: EC2와 RDS가 같은 VPC로 연결되지 않은 상태였음
- **해결**: EC2를 RDS와 같은 VPC에 연결 → 5432 포트 TCP 연결 확인(`/dev/tcp` 테스트) 후 정상화

### Docker Compose 빌드 실패 (`compose build requires buildx 0.17.0 or later`)
- **원인**: Amazon Linux 2023 기본 docker에 포함된 buildx가 0.12.1로 너무 오래됨
- **해결**: GitHub 릴리스에서 buildx v0.35.0 바이너리를 직접 받아 `/usr/local/lib/docker/cli-plugins/`에 설치

### Google OAuth 리다이렉트 거부 — "공개 최상위 도메인으로 끝나야 합니다"
- **원인**: Google OAuth는 보안 정책상 **IP 주소를 리다이렉트 URI로 허용하지 않음** (도메인 필수)
- **해결**: `nip.io` 무료 와일드카드 DNS 사용 — `13.211.170.219` → `13.211.170.219.nip.io`로 그대로 매핑됨, 비용 없이 즉시 적용
- 변경 사항: Google Console 리다이렉트 URI, `SecurityConfig`의 CORS `allowedOriginPatterns`에 `*.nip.io`, `*.cloudfront.net` 패턴 추가, EC2 `.env`의 `FRONTEND_URL`

### EC2 메모리 부족으로 빌드 멈춤 (응답 없음)
- **증상**: `docker compose up -d --build`가 `mvnw package` 단계에서 멈춘 채 진행이 안 됨. 이후 SSH 새 연결조차 타임아웃
- **원인**: t3.micro는 RAM 1GB뿐인데, 이미 떠 있는 backend+ai-service 컨테이너가 메모리를 점유한 상태에서 Maven 빌드(JVM)를 추가로 돌리며 메모리 고갈 → swap이 없어서 OOM 직전까지 감
- **해결**:
  1. AWS 콘솔에서 EC2 재부팅 (멈춘 프로세스 정리)
  2. 재부팅 후 SSH도 안 됨 → 원인은 보안 그룹 SSH(22) 인바운드가 "내 IP" 고정이었는데 집 IP가 바뀌어서 막혔던 것. 보안 그룹 소스 IP 갱신으로 해결
  3. **2GB swap 파일 생성** (`fallocate` + `mkswap` + `swapon`) 후 `/etc/fstab`에 등록해 재부팅 후에도 유지되게 설정
  4. swap 추가 후 재빌드 시 멈춤 없이 16초 내 정상 완료
